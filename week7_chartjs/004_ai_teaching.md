# 與 AI 協作：視覺化大師篇 (Week 7)

要背熟 Chart.js 那些用來調顏色、調線條平滑度、調工具框字體的設定檔，真的太痛苦了。這就是 Vibe Coding 最大的優勢——讓 AI 幫你寫這種繁瑣的「外觀設定」。

## 💡 AI 提示詞 (Prompt) 技巧分享

對於畫圖，你必須明確交代：資料長什麼樣、圖表類型、以及你對「美感」的具體要求。

### ✅ 好的 Prompt 範例

> **你現在是一個專業的前端資料視覺化工程師。**
> 我的專案使用的是 **Chart.js** 搭配 Jinja2 模板引擎與 Bootstrap 5。
>
> 請幫我在網頁的 `<div>` 容器中加上一個 `<canvas>` 畫布，並寫出對應的 JavaScript 來繪製一張**折線圖 (Line Chart)**，用來展示台積電近一個月的股價走勢。
>
> **前後端整合設定：**
> - 我的 X 軸日期陣列會透過 Jinja2 的 `{{ date_labels | safe }}` 動態傳入。
> - 我的 Y 軸股價陣列會透過 `{{ price_values | safe }}` 動態傳入。
>
> **圖表的外觀要求：**
> 1. 請將這條線設定為代表上漲的「綠色」，線條要平滑曲線 (`tension: 0.3`)。
> 2. 請將這條線底下的面積填上一點淡淡的半透明綠色漸層 (使用 `fill: true`)，增加質感。
> 3. 不要顯示每個資料點的圈圈，只在游標 hover 過去的時候才顯示圈圈 (`pointRadius: 0`)。
> 4. X 軸跟 Y 軸的網格線 (Grid lines) 請設定為非常淡的灰色，看起來比較不凌亂。
>
> 給我需要的 HTML (canvas 區塊) 與完整的 JavaScript 區塊 (`<script>...`)。

### 🚨 常見的 AI 翻車踩坑點

1. **Jinja2 安全濾鏡遺失 (致命錯誤)**：
   部分 AI 寫 Jinja2 給 JS 的時候，會傻傻地只寫 `const myData = {{ my_data }};`。這就是我們剛剛上一章提到的，這會導致字串出現 `&#39;`，圖表直接死亡！
   **解決方法**：如果 AI 給的圖表不出來，打開瀏覽器的主控台 (F12 -> Console) 看是不是報錯，並堅定地跟 AI 說：「請在 Jinja 變數後面加上 `| safe` 避免字元被編碼。」

2. **畫布無限長大 (Canvas Resize Issue)**：
   HTML 的 `<canvas>` 元素有個很搞怪的特性，如果外面的容器沒有設定好，它有時候會隨著視窗一直無限長大，最後破壞版面。
   **解決方法**：Chart.js 建議畫布外面要包一個 `<div>`。你可以要求 AI：「請給我的 canvas 包一層具備 `position: relative` 屬性的 div 容器，以確保圖表 RWD 響應式能正常運作。」

3. **忘記載入 Chart.js**：
   記得一定要在 HTML `<head>` 區塊透過 CDN 載入套件。AI 寫到一半經常忘記給你 CDN 連結，你可以自己去拿，或提醒它：「請順便提供 Chart.js 的 CDN」。
