# 與 AI 協作：前端設計篇 (Week 6)

對大部份後端或資料背景的工程師來說，前端 UI 設計通常是最頭痛的。好消息是，AI 寫 Bootstrap 網頁的能力也是頂尖的！但如果隨便叫 AI 生一個網頁，它常常會生出很陽春、甚至是十年前風格的排版。

## 💡 AI 提示詞 (Prompt) 技巧分享

要請 AI 產生具備「現代感」與「專業感」的財經網頁，你的 Prompt 必須要給出具體的「版面配置」和「情境」。

### ✅ 好的 Prompt 範例

> **你現在是一個專業的前端 UI 設計師。**
> 請幫我用 HTML 與 **Bootstrap 5** (請透過 CDN 引入) 寫一個「財經價值投資儀表板」的頁面。
> 
> **請照著這個版面結構來設計：**
> 1. **最上方**：固定在頂部的深色導航列 (Navbar)，標題是「護城河投資儀。
> 2. **上半部**：使用 3 個小卡片 (Cards)，排在同一列。分別顯示「大盤指數」、「今日亮點股票」、「系統收錄檔數」。卡片請加上微小的陰影 (`shadow-sm`)。
> 3. **下半部**：一個全寬的區塊，裡面放一個美觀的資料表格 (Table)。
>    - 表格要有斑馬紋 (striped) 和 hover 效果。
>    - 表格的欄位為：股票代號、名稱、最新股價、本益比、操作。
> 
> **特別要求：**
> - 表格內的資料請先寫死 (Mock Data) 2筆示範資料就好。
> - 整體設計請保持「極簡、現代、財經專業感」，適當使用邊距 (`p-3`, `m-2` 等)。
> - 在需要填入後端動態變數的地方，請用 `{{ }}` 像是 Jinja2 的語法做標記，方便我之後跟 FastAPI 整合。
>
> 請一次給我完整的 `index.html` 內容。

### 🚨 常見的 AI 翻車踩坑點

1. **版本錯亂**：
   Bootstrap 現在的主流是第 5 版 (v5)，它不用依賴 jQuery。但有些 AI 如果沒有特別指定，可能會給你第 3 版或第 4 版的舊語法，導致你複製貼上卻發現版面跑版。
   **解決法**：在 Prompt 裡永遠強調「請使用 Bootstrap 5」。

2. **Jinja2 語法破壞**：
   有時候 AI 用 Jinja2 的 `{% for %}` 迴圈時，放的位置不對 (例如放在 `<table>` 外面，或是 `<tr>` 外面但裡面忘記把 `<tr>` 包進去)，這會導致網頁顯示出長得很畸形的表格。
   **防禦機制**：請 AI 寫完後，自己肉眼檢查一下，迴圈 `{% for ... %}` 是不是剛剛好包住了那一行 `<tr>` ... `</tr>`，確保每一筆資料都能產生換行。

3. **忘記靜態資源 (Static Files)**：
   如果是請 AI 寫自己準備的 CSS (非 Bootstrap)，你要記得在 FastAPI 端設定 `StaticFiles` 的路由。為了降低一開始的複雜度，我們這週強烈建議都使用直接透過網路引入資源 (CDN) 的方式套用 Bootstrap。
