# Week 4: 課程內容 (Course Content)

## 學習目標

在 Week 3 中，我們學會了如何定義自己專屬的股票資料庫。但是，現在資料庫裡面空無一物！我們總不可能一檔一檔股票自己手動打進去吧？本週的目標是學習如何自動連線到真實的金融世界，把 Yahoo Finance 的即時數據與歷史股價，自動抓下來並存進我們的 SQLite 資料庫裡。

## 涵蓋主題

1. **金融數據來源簡介**
   - 為什麼選擇 Yahoo Finance (透過 `yfinance` 套件)？
   - 認識其他潛在的數據源 (例如：TwStock、富邦 API、以及付費的 Bloomberg/Reuters 等差異)。

2. **yfinance 套件實戰**
   - 安裝與基本設定。
   - 如何抓取特定股票 (例如：`AAPL` 或台股 `2330.TW`) 的即時價格與基本面資料 (PE Ratio, 市值等)。
   - 如何抓取歷史的 K 線資料 (開高低收、成交量)。

3. **初探 ETL (Extract, Transform, Load)**
   - 資料工程師的日常：ETL 是什麼？
   - **Extract (萃取)**：從 Yahoo API 抓下 JSON 或是 Pandas DataFrame 資料。
   - **Transform (清理轉換)**：把抓下來的髒資料過濾掉 (例如把 NaN 補成 0，或過濾掉國定假日的空資料)。
   - **Load (載入)**：用我們 Week 3 學到的 SQLModel，把整理好的乾淨資料存進資料庫裡永久保存。

## 本週預期產出

- 成功執行 Python 腳本，把某檔股票最新的一筆股價與本益比列印在終端機。
- 完成一個能自動化執行的「資料更新小程式」：輸入幾檔有興趣的股票代號，它就會自己去網路抓最新資料，然後寫進你本地的 SQLite 資料庫做更新。
