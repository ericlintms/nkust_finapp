# 實務範例與實作 (Week 4)

上週我們體驗了怎麼自己用手敲指令存資料到資料庫，今天我們要讓程式「全自動」幫我們到網路上運貨回家！

## 我們的練習清單

### 1. 例題一：和 Yahoo Finance 第一次親密接觸

- **對應檔案**: `examples/w401_fetch_price.py`
- **練習目的**: 學會如何用 Python 的 `yfinance` 套件對特定的股票做出查詢。
- **實作任務**:
  1. 切換目錄到 `week4_yfinance/examples/`。
  2. 確保你安裝了套件：在終端機輸入 `pip install yfinance`。
  3. 執行程式：`python w401_fetch_price.py`。
  4. 觀察終端機是不是順利印出了台積電的資訊。
  5. 試著打開程式碼，把 `2330.TW` 換成 `NVDA` 或你喜歡的美股，看看能抓出什麼神奇的數字。

### 2. 例題二：打造你的第一條微型 ETL 產線

- **對應檔案**: `examples/w402_save_to_db.py`
- **練習目的**: 把 Week 3 的資料庫與 Week 4 的爬蟲結合起來！
- **實作任務**:
  1. 在執行這支程式前，我們要確保有上週的 `database.db` 還有那幾支設定檔案。為了方便，我把 `create_db.py` 這類的基礎建設直接寫在範例內部了。
  2. 執行程式：`python w402_save_to_db.py`。
  3. 看看終端機，你會發現它會先跑一趟網路 (Extract)、處理資料 (Transform)、最後啟動資料庫推車把它存進去 (Load)。
  4. 此時你的 `database.db` 已經擁有了真實世界的即時資訊。

---

> 💡 **小提示 (Hint for students)**：
>
> 為什麼我們要把「抓資料」(Extract) 和「存資料」(Load) 寫成不同的函數分開來？
> 這也是業界最重要的軟體工程思維之一：**單一職責原則 (Single Responsibility)**。
> 去網路要資料的人就是去要資料，他不需要知道資料庫保險箱的密碼。
> 把職責拆開，如果你哪天覺得 Yahoo 資料太慢，想換成另一個網站，你只要改 Extract 函數就好，不用連資料庫存檔的邏輯都重寫！
