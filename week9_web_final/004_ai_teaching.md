# 與 AI 協作：程式碼打磨與重構 (Week 9)

這週的作業是「大掃除」！把這八週以來東抄一點、西抄一點的散落程式碼組合起來，這也是 AI 發揮統整能力的最佳時機。

在軟體工程裡，重新整理程式碼讓他更漂亮、更穩定，而不改變原本的功能，我們稱之為 **重構 (Refactoring)**。

## 💡 AI 提示詞 (Prompt) 技巧分享

當你的 Python 檔案暴增到 300 行時，就該請 AI 幫你重構了。

### ✅ 好的 Prompt 範例

> **你現在是一個注重程式碼品質的高級軟體工程師 (Senior Software Engineer)。**
> 
> 我目前有一個 FastAPI 的 Python 檔案，裡面包含了資料庫存取 `SQLModel`、資料抓取爬蟲 `yfinance`、以及路由運算邏輯，由於全部寫在一起，變得非常難以維護。
> 
> *(附上你的完整程式碼)*
> 
> 請幫我進行「重構 (Refactor)」，要求如下：
> 1. 將**資料庫定義**拆分到獨立的 `models.py` 檔案。
> 2. 將**爬蟲資料抓取邏輯**拆分到獨立的 `services.py` 檔案。
> 3. 主檔案 `main.py` 只保留 FastAPI 的路由 (Router) 與依賴注入 (Dependency Injection)。
> 4. 在這個過程中，如果發現有沒有寫 Try-Except 錯誤處理的地方，請一併幫我補上。
> 5. 請保持變數命名風格一致，並補齊 Type Hints (例如 `def fetch_data(ticker: str) -> dict:`)。
>
> 請提供我這三個檔案拆分後的完整內容。

### 🚨 常見的 AI 翻車踩坑點

1. **AI 改太嗨，改變了原有邏輯**：
   有時候請 AI 重構，它因為想要展現高超的技術，會把你原本簡單易懂的 `if/else` 改成非常難懂的列表推導式 (List Comprehension) 或 Lambda 函數。
   **解決方法**：在提示詞加上：「請專注於檔案結構拆分與增強防護，**切勿改變我原本寫好的核心數學運算邏輯**。」

2. **忽略了 import 的關聯性 (ModuleNotFoundError)**：
   當 AI 把大檔案拆成三個小檔案後，經常會忘記在檔案開頭把互相需要的套件 `import` 進來（例如 `main.py` 忘了 `from models import Stock`）。
   **防禦機制**：拆分完成後，第一次執行程式一定會跳很多找不到套件的紅字，不用慌張，冷靜地把報錯訊息貼還給 AI 叫它補上 import 即可。
