# 實務範例與實作 (Week 3)

這週的練習，我們要真正把資料永久存進硬碟裡！
請到 `examples/` 資料夾底下打開這些程式碼來練習。執行這些範例後，你會看到旁邊多出一個真實的 `.db` 檔案喔！

## 我們的練習清單

### 1. 例題一：建立我們的第一個資料表

- **對應檔案**: `examples/w301_create_db.py`
- **練習目的**: 了解怎麼告訴 SQLModel，幫我們把定義好的 Class 轉換成硬碟裡的資料表。
- **實作任務**:
  1. 在終端機執行 `cd examples/`，確保你在正確的目錄。
  2. 執行 `python w301_create_db.py`。
  3. 觀察終端機的輸出，看看 SQLModel 是怎麼在背後幫我們產生一長串的 SQL 指令 (`CREATE TABLE ...`)。
  4. 確認資料夾中是不是多出了一個 `database.db` 的檔案？這就是你的心血結晶！(你可以去下載 DB Browser for SQLite 這種軟體來打開它看看裡面)。

### 2. 例題二：CRUD 終極大挑戰

- **對應檔案**: `examples/w302_crud_stock.py`
- **練習目的**: 熟悉最常用的四個資料庫操作：Create (新增)、Read (讀取)、Update (更新)、Delete (刪除)。這四個字合起來就是著名的 **CRUD**。
- **實作任務**:
  1. 執行 `python w302_crud_stock.py`。
  2. 觀察程式執行的每一個步驟印出來的訊息。
  3. **動手改改看**：試著打開這個檔案，把裡面的台積電換成聯發科 (2454.TW)，價格自己定，然後重新執行一次程式，看看邏輯是不是跟著改變了！

---

> 💡 **小提示 (Hint for students)**：
>
> 仔細看程式碼裡面的 `Session(engine)`。Session 中文叫「工作階段」，它就像是資料庫的「推車」。
> 當你要新增資料時，你是把物品放進推車 (`session.add()`)。
> 但資料這時候還沒真正結帳！只有當你呼叫了 `session.commit()`（結帳！），資料才會安安穩穩地寫進 `.db` 檔案裡面。千萬別忘了結帳啊！
