# Week 3: 課程內容 (Course Content)

## 學習目標

上週我們學會了用 FastAPI 建立可以接收請求的服務，但我們的資料都是算完就消失了！對一個真正的財經應用程式來說，我們必須要把資料「持久化」存下來。這週我們將學習如何使用關聯式資料庫 (SQLite) 以及 Python 的 ORM 神器 (SQLModel)，打造能永久保存股票與投資組合資料的基礎。

## 涵蓋主題

1. **關聯式資料庫與 SQLite 基礎**
   - 什麼是資料庫？這跟我們常用的 Excel 有什麼不一樣？
   - 為什麼選擇 SQLite？輕量級、免安裝伺服器的完美開發首選。
   - 資料表 (Table) 的結構概念：資料列 (Rows)、資料行 (Columns) 與主鍵 (Primary Key)。

2. **了解 ORM (物件關聯對應) 與 SQLModel**
   - 為什麼不用手寫 SQL 語法？ORM 的魔力。
   - 介紹 SQLModel：結合了 Pydantic (資料驗證) 與 SQLAlchemy (資料庫操作) 的現代化武器。
   - 如何把 Python 的 Class (類別) 變成資料庫裡的 Table (資料表)。

3. **實踐 CRUD 操作**
   - 建立資料庫連線 (Engine & Session)。
   - **C**reate: 新增一檔股票資料 (例如台積電)。
   - **R**ead: 查詢特定的股票，或是一次把所有清單抓出來。
   - **U**pdate: 當股票價格或 PE 值改變時，如何更新資料。
   - **D**elete: 刪除下市或不再追蹤的股票。

## 本週預期產出

- 成功在自己的專案目錄下自動生成一個 `.db` 資料庫檔案。
- 寫出一個 Python 程式碼，成功地將特定股票的代號、價格、本益比 (PE) 存進資料庫。
- 能夠用 Python 查詢出剛剛存進去的資料，並在終端機印出來證明它真的存在！
