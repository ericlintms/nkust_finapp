# Week 1: 理論解說 (Theory Explanation)

## 1. 貨幣的時間價值 (Time Value of Money)

我們財管系最核心的概念就是：「今天的 100 元，比明天的 100 元更有價值」。這是因為錢可以投資生息，而且通膨也會讓錢變薄。在這個基礎上，我們這週要實作淨現值 (NPV) 與內部報酬率 (IRR)。

### 淨現值 (NPV, Net Present Value)

把未來每期會產生的現金流 (Cash Flow)，用一個特定的**折現率 (Discount Rate)** 換算回「現在的價值」，然後全部加總起來。

- 完整公式：`NPV = CF₀ + Σ[t=1..n] [ CFt / (1 + r)^t ]`，其中 `CF₀` 是**第 0 期（期初）的投資金額（通常是負數）**，`CFt` 是第 `t` 期的現金流，`r` 是折現率。
- **⚠️ 常見陷阱**：期初投入（t=0）已經是「現在的錢」，所以**不需要折現**，直接加進來就好。未來每一期的現金流（t=1, 2, 3...）才需要除以 `(1+r)^t` 折現回來。這個細節 AI 很容易搞錯，Week 1 就要記住！
- 決策法則：只要算出來的 `NPV > 0`，就代表這項投資的總價值大於原本投入的成本，那我們就應該接受這個案子。

```mermaid
flowchart TD
    A["🏦 期初投入資金<br/>(第 0 期，通常為負值)"] --> B["📅 未來各期現金流<br/>CF₁, CF₂, CF₃ ..."] 
    B --> C["📉 各期現金流 ÷ (1 + r)^t<br/>折現回現在的價值"]
    C --> D["➕ 全部折現值加總"]
    D --> E{NPV 結果判斷}
    E -- "NPV > 0" --> F["✅ 投資划算，接受專案"]
    E -- "NPV = 0" --> G["⚖️ 剛好打平，視情況決定"]
    E -- "NPV < 0" --> H["❌ 投資不划算，拒絕專案"]
```

### 內部報酬率 (IRR, Internal Rate of Return)

我們可以把 IRR 想像成一個特別的折現率，它能夠讓這項投資的 `NPV 剛好變成 0`。

- 意義：算出來的數字也就是這項專案自己本身的「年化投資達成率」。
- 決策法則：一旦 `IRR > 我們要求的最低回報率 (Hurdle Rate)`，這項投資就是划算的。

## 2. 什麼是 Vibe Coding？

過去資工系寫程式是「自己一行一行手刻代碼」，而我們這門課要學的 Vibe Coding 則是把 AI (像是 ChatGPT 或 LLM) 當成我們的專屬工程師：

- **開發者的角色轉變**：我們不再只是個寫 code 的「打字員」，而是晉升成設計系統架構的「專案經理」。
- **工作流程**：我們負責設定大框架跟規則 -> 給 AI 下達清晰具體的指令 (Prompt) -> AI 幫我們生成程式碼 -> 我們自己去驗證它算得對不對。
- 在跟數字打交道的財務開發中，「驗證邏輯」絕對是最重要的一環，因為那些生出來的金融數字絕對不能出錯。

```mermaid
flowchart LR
    A["🎯 我們<br/>(財管系學生/專案經理)"] -->|"1. 設定需求\n& 下 Prompt"| B["🤖 AI<br/>(專屬工程師)"]
    B -->|"2. 生成程式碼"| C["📄 Python 程式碼"]
    C -->|"3. 執行驗算"| D{"✅ 結果<br/>正確嗎？"}
    D -->|"Yes → 完成"| E["🚀 部署上線"]
    D -->|"No → 告訴 AI 哪裡錯"| A
```
